# 📝 Next.js, SQLite & Tailwind CSS 기반 Todo 웹앱 제품 요구사항 명세서 (PRD)

## 1. 개요 (Overview)
*   **제품명:** 오늘의 할 일 (Next.js Todo App)
*   **목적:** 사용자가 할 일을 등록, 관리, 완료 처리할 수 있는 풀스택(Full-Stack) 웹 애플리케이션입니다.
*   **핵심 특징:** Next.js와 SQLite로 백엔드를 구축하고, **Tailwind CSS**를 사용하여 깔끔하고 트렌디한 디자인의 화면을 신속하게 완성합니다.

---

## 2. 기술 스택 (Technology Stack)
*   **웹 프레임워크:** **Next.js (App Router)**
    *   *설명:* 프론트엔드 화면 구성과 백엔드 API 서버를 한 번에 다룰 수 있는 풀스택 프레임워크입니다.
*   **스타일링 (UI):** **Tailwind CSS (v3)**
    *   *설명:* CSS 파일을 따로 두지 않고, HTML 태그 내 클래스명 조립으로 신속하게 반응형 UI를 디자인하는 유틸리티 우선 CSS 프레임워크입니다.
*   **데이터베이스 (DB):** **SQLite (임시 로컬 DB)**
    *   *설명:* 복잡한 설정 없이 파일 하나(`.db`)로 모든 데이터를 저장하는 경량 파일형 데이터베이스입니다.
*   **데이터베이스 연결 도구 (ORM):** **Prisma**
    *   *설명:* 데이터베이스 코드를 쉽게 짜도록 돕는 번역기입니다. 나중에 클라우드 DB인 Neon DB로 이전할 때 코드 수정을 최소화해 줍니다.
*   **향후 전환 대상 DB:** **Neon DB (PostgreSQL)**
    *   *설명:* 추후에 실제로 배포할 때 사용할 클라우드 기반 PostgreSQL 데이터베이스입니다.

---

## 3. 시스템 아키텍처 및 데이터 흐름 (Architecture)
사용자가 버튼을 누를 때 데이터가 흘러가는 구조입니다.

```
[ 브라우저 (클라이언트) ] 
        │  (1) 할 일 추가 요청 (HTTP POST /api/todos)
        ▼
[ Next.js 백엔드 서버 (API Routes) ]
        │  (2) Prisma ORM을 거쳐 SQL 쿼리로 번역
        ▼
[ SQLite 데이터베이스 (local.db 파일) ] <-- 나중에 Neon DB로 쉽게 교체 가능
```

## 4. 기능적 요구사항 (Functional Requirements)

### A. 할 일 추가 (Create)
*   **사용자 기능:** 입력창에 할 일을 적고 '추가' 버튼을 누르거나 `Enter` 키를 눌러 할 일을 등록합니다.
*   **서버 처리:** 백엔드 API에서 요청을 받아 SQLite 데이터베이스에 레코드를 저장합니다.
*   **데이터베이스 테이블 구조 (Schema):**
    *   `id` (Integer, Primary Key): 고유 번호
    *   `text` (String): 할 일 내용
    *   `completed` (Boolean): 완료 여부 (기본값: `false`)
    *   `createdAt` (DateTime): 생성 시간
*   **예외 처리:** 빈 입력값은 전송되지 않도록 프론트엔드 및 백엔드 양측에서 유효성 검사를 합니다.

### B. 할 일 조회 및 필터링 (Read)
*   **사용자 기능:** 저장된 할 일 목록을 화면에 리스트 형식으로 출력합니다.
*   **필터링 기능:**
    1.  **전체(All):** 모든 할 일 출력
    2.  **진행 중(Active):** 아직 완료되지 않은(`completed: false`) 할 일만 출력
    3.  **완료됨(Completed):** 완료 처리된(`completed: true`) 할 일만 출력

### C. 할 일 완료 및 수정 (Update)
*   **완료 체크:** 체크박스를 누르면 DB의 `completed` 상태가 반전(`true` <-> `false`)되고, 완료된 일에는 취소선이 그어집니다.
*   **내용 수정:** 할 일 텍스트를 더블클릭하면 텍스트 박스로 변경되어 내용을 수정 및 저장할 수 있습니다.

### D. 할 일 삭제 (Delete)
*   **개별 삭제:** 할 일 오른쪽 끝의 '삭제(X)' 버튼을 누르면 DB에서 삭제됩니다.
*   **일괄 삭제:** '완료된 항목 전체 삭제(Clear Completed)' 버튼으로 완료된 항목들을 한 번에 지웁니다.

## 5. 비기능적 요구사항 (Non-Functional Requirements)
*   **반응형 웹 디자인:** 모바일(화면 꽉 차게)과 PC(중앙 정렬 및 정돈된 레이아웃) 환경을 모두 지원합니다.
*   **DB 이식성:** 데이터베이스 연결 정보는 환경 변수(`.env`) 파일에 정의하여, SQLite에서 Neon DB로 이식할 때 설정만 변경하도록 설계합니다.
*   **일관된 컬러 테마:** 사용자 경험(UX)을 향상하기 위해 Tailwind CSS의 일관된 팔레트를 사용하여 시각적 만족도를 높입니다.
